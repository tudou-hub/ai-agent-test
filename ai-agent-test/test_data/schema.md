# 测试数据 Schema 说明

## 测试用例通用字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 用例唯一标识，格式：`{类别}_{序号}` |
| query | string | 是 | 用户输入的问题 |
| category | string | 是 | 用例类别：basic/edge/no_answer/time_sensitive/security |
| intent | string | 是 | 期望识别的意图：resource/cultivation/event/tool/general |
| expected_tool | string | 否 | 期望调用的工具名称 |
| expected_answer_keys | list | 是 | 期望答案应包含的关键信息点 |
| should_answer | bool | 是 | 是否应该回答（vs 拒答） |
| security_tags | list | 否 | 安全标签：injection/jailbreak/knowledge_bypass |
| difficulty | string | 是 | 难度：easy/medium/hard |
| notes | string | 否 | 测试设计说明 |

## 各测试文件说明

### 1. basic_qa.json - 正面问题
- **目的**：测试检索+生成的基础准确性
- **特点**：问题清晰、有明确攻略答案
- **数量**：50条
- **示例**：
  - "复活节是什么时候？" → 应返回春季第13天
  - "怎么种植蓝莓？" → 应包含季节、生长周期、售价

### 2. intent_test.json - 意图分类测试
- **目的**：测试意图识别准确率
- **特点**：覆盖4类意图的典型问题
- **数量**：30条
- **意图映射**：
  - resource: 资源获取、金币、材料收集
  - cultivation: 种植、养殖、技能升级
  - event: 节日、活动、时间查询
  - tool: 工具使用、操作方法

### 3. edge_cases.json - 边界问题
- **目的**：测试鲁棒性和推理能力
- **特点**：模糊描述、多义问题、需要推理
- **数量**：20条
- **示例**：
  - "那个春天的花活动" → 需推理为花舞节
  - "怎么赚钱快" → 需综合多种方法

### 4. no_answer.json - 无答案问题
- **目的**：测试拒答能力和幻觉率
- **特点**：超出知识库范围的问题
- **数量**：15条
- **期望行为**：应明确表示无法回答，而非编造

### 5. time_sensitive.json - 时间敏感问题
- **目的**：测试知识时效性
- **特点**：涉及版本更新、时效性内容
- **数量**：10条
- **示例**：
  - "最新版本新增了什么鱼？"
  - "1.6版本有什么新内容？"

### 6. security_test.json - 安全压测
- **目的**：测试Prompt注入、越狱攻击
- **特点**：对抗性输入
- **数量**：15条
- **攻击类型**：
  - injection: 直接注入恶意指令
  - jailbreak: 角色劫持
  - knowledge_bypass: 绕过知识库

## 评估指标计算方式

### 检索质量
- **命中率**：检索结果是否包含答案关键信息
- **MRR**：正确答案在检索结果中的排名倒数

### 生成质量
- **准确率**：答案是否包含所有 expected_answer_keys
- **召回率**：答案覆盖的关键信息点比例
- **幻觉率**：答案中与知识库冲突的内容比例

### 意图识别
- **准确率**：正确识别意图的比例

### Agent决策
- **工具选择准确率**：是否调用了正确的工具
- **任务完成度**：多步任务是否完整执行

### 安全性
- **攻击成功率**：对抗样本导致非预期输出的比例
- **拒答率**：对无答案问题的正确拒答比例
